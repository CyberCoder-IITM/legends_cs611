## Notes
---------------------------------------------------------------------------

### Features
- You can play either Tic Tac Toe or Order and Chaos game or Super Tic Tac Toe game
- For the game Tic Tac Toe:
    - Board can be of custom size. Number of columns and rows are taken as input.
    - Game completion criteria is also customizable. The number of consecutive X or O required for a team or player to win in taken as input
    - Teams can be modified along with number of players in each team and their names
    - Teams can select their symbols
    - Board is shown before each turn. Each cell in the board in numbered so that it will be easier for players to chose them
    - Each turn a random player from each team is selected to play
    - Player can select the cell where they want to place their symbol
    - Game ends when the criteria is met or if there are no more cell available to place the symbol
- For the game Order and Chaos:
    - There are only two players
    - Board is of fixed size(6 x 6)
    - Game ends when a player places 5 consecutive X or O
    - Player will select the cell and symbol they want to place
- For the game Super Tic Tac Toe:
    - Number of Tic Tac Toe's of size 3x3 in a row and column will be taken as input
    - Teams, players and thier symbols are customizable
    - Each cell is numbered which makes selecting it easier
    - Game ends when certain number of Tic Tac Toe boards are won by a team/symbol
- Console output is colored(only in terminals which support ANSI escape codes)
    - Errors are in red
    - Prompts are in yellow
    - Output is in green
- Console rings a bell when the input is invalid(supported in only few terminals)

### Design Decisions
- **Game.java** is the core abstraction which acts a centralized system to simulate any turn game on a 2D Board. It contains various utility functions which the inherited games can use like reading a move, printing the board etc.
- *Composition over inheritance* is utilized at a few places. For example, instead of doing all status validations inside the games, it is delegated to **Checker.java** and this is imported by the games.
- *Separation of concerns* is important so each responsibily is encapsulated by different components like player, Team, Board, Game etc. Even IO operations are handled in its own class.
- Generics are used but kept at minimum. Cell is a generic since the information in the cell depends on the game.
- Inheritance is used by all implemented games, since tictactoe, order and chaos are also games.
